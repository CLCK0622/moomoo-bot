"""
配置文件说明 - 详细注释版本

本文件详细说明了 config.py 中各个参数的含义和调整建议
"""

# ==================== Moomoo OpenD 连接配置 ====================
"""
MOOMOO_HOST: OpenD 服务地址
- 默认: '127.0.0.1' (本地运行)
- 如果 OpenD 运行在其他机器，修改为对应 IP

MOOMOO_PORT: OpenD 端口
- 默认: 11111
- 如果修改了 OpenD 配置，需要同步修改

MARKET: 市场代码
- 'US': 美股
- 'HK': 港股
- 'CN': A股

TRADE_ENV: 交易环境
- 0: 真实交易（请谨慎！）
- 1: 模拟盘（建议先用模拟盘测试）

SECURITY_FIRM: 券商
- 2: Moomoo SG (新加坡)
- 1: Moomoo HK (香港)
- 其他值参考 futu-api 文档
"""

# ==================== 交易时段 ====================
"""
MARKET_OPEN_TIME: 开盘时间
- 美东时间 09:30

MARKET_CLOSE_TIME: 收盘时间
- 美东时间 16:00

ORB_END_TIME: ORB 计算完成时间
- 美东时间 09:45
- 这是策略开始监控入场信号的时间

FORCE_CLOSE_TIME: 强制平仓时间
- 美东时间 15:55
- 在收盘前 5 分钟强制平掉所有持仓
"""

# ==================== 资金管理参数 ====================
"""
MAX_POSITIONS: 最大持仓数量
- 默认: 5 只股票
- 建议范围: 3-8
- 太少：分散不够
- 太多：资金过于分散，单笔收益小

POSITION_SIZE_RATIO: 每只股票占比
- 默认: 0.2 (即 1/5)
- 计算公式: 1 / MAX_POSITIONS

MARGIN_FREEZE_RATIO: 市价单冻结比例
- 默认: 1.2 (即 20%)
- Moomoo 市价单会冻结额外资金，需要除以这个比例
- 实际可用资金 = 账户总资金 * POSITION_SIZE_RATIO / MARGIN_FREEZE_RATIO
"""

# ==================== 技术指标参数 ====================
"""
ATR_PERIOD: ATR 周期
- 默认: 14
- 建议范围: 10-20
- 周期越短，对波动越敏感
- 周期越长，对波动越平滑

KC_EMA_PERIOD: Keltner Channel 的 EMA 周期
- 默认: 20
- 建议范围: 15-30
- 这是 KC 的中轨（基准线）

KC_ATR_MULTIPLIER: KC 带宽倍数
- 默认: 2.0
- 建议范围: 1.5-3.0
- 倍数越大，通道越宽，信号越少但可靠性越高
- 倍数越小，通道越窄，信号越多但误报率越高
"""

# ==================== 止损止盈参数 ====================
"""
STOP_LOSS_RATIO: 账户止损比例
- 默认: 0.03 (3%)
- 建议范围: 0.02-0.05
- 单只股票浮亏达到此比例立即清仓
- 这是最后的风险保护线

TP1_REDUCE_RATIO: 第一目标止盈减仓比例
- 默认: 0.5 (50%)
- 建议范围: 0.3-0.7
- 达到 TP1 后卖出多少仓位
- 0.5 表示卖出一半，留一半继续跑

TP2_ATR_MULTIPLIER: 第二目标的 ATR 倍数
- 默认: 3.0
- 建议范围: 2.5-4.0
- TP2 = KC中轨 + TP2_ATR_MULTIPLIER * ATR
- 倍数越大，目标越远，达到概率越低

TREND_REVERSAL_BARS: 趋势反转确认的 K 线数量
- 默认: 2 根
- 建议范围: 1-3
- 连续 N 根 15 分钟 K 线跌破 KC 中轨才确认反转
- 1 根：反应快，但可能误判
- 3 根：确认充分，但可能反应慢
"""

# ==================== 数据频率 ====================
"""
BAR_1M: 1 分钟 K 线
- 用途: 计算 ORB、VWAP、实时价格监控

BAR_15M: 15 分钟 K 线
- 用途: 计算 KC、ATR、趋势确认
"""

# ==================== 状态机 ====================
"""
STATE_IDLE (0): 空仓状态
- 可以接收新的开仓信号

STATE_POSITION (1): 持仓中（初始仓位）
- 已开仓，等待 TP1 或止损

STATE_HALF_PROFIT (2): 半仓止盈后
- TP1 已触发，减仓 50%，等待 TP2 或趋势反转

STATE_DONE (3): 当日结束
- 已清仓或达到最大交易次数，当日不再交易此股票
"""

# ==================== 调参建议 ====================
"""
保守型参数（降低风险）:
- STOP_LOSS_RATIO = 0.02 (2% 止损)
- KC_ATR_MULTIPLIER = 2.5 (更宽的通道，信号更少)
- TP1_REDUCE_RATIO = 0.7 (达到 TP1 卖出 70%)
- TREND_REVERSAL_BARS = 2 (更谨慎的反转确认)

激进型参数（追求收益）:
- STOP_LOSS_RATIO = 0.05 (5% 止损，给更多空间)
- KC_ATR_MULTIPLIER = 1.5 (更窄的通道，信号更多)
- TP1_REDUCE_RATIO = 0.3 (达到 TP1 只卖出 30%)
- TREND_REVERSAL_BARS = 3 (给趋势更多时间)

回测建议:
1. 先用默认参数运行 1-2 周
2. 记录每笔交易的开仓原因、平仓原因
3. 分析哪些参数导致了盈利/亏损
4. 小步调整，一次只改 1-2 个参数
5. 对比调整前后的表现
"""
